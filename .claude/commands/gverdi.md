## Mission

როდესაც თამარი იყენებს `/gverdi`-ს (ახალი გვერდის ან ცვლილების დავალებით), ეს ნიშნავს: სამუშაო უნდა შესრულდეს ვებსაიტის დიზაინ-სისტემის **ზუსტი დაცვით**, არა ინტუიციით ან ნულიდან.

### ნაბიჯი 1: წაიკითხე დიზაინ-სისტემა
სანამ რამეს გააკეთებ, წაიკითხე `livrables/site-web/DESIGN-SYSTEM.md` **სრულად**. ეს არის ერთადერთი წყარო.

### ნაბიჯი 2: ააგე/შეცვალე წესების მიხედვით
- **`<head>`** — boilerplate (სექცია 15): ზუსტი stylesheet-ები + Google Fonts.
- **სტრუქტურა** — skeleton (სექცია 16): NAV (verbatim) → hero → content (article სისტემა) → Newsletter (verbatim) → Footer (verbatim) → `site.js`.
- **ტიპოგრაფია** — მკაცრი (სექცია 12): სათაური serif, ტექსტი sans 17px (article 16px), meta mono. ზუსტი ზომები, გადახვევის გარეშე.
- **ფერები** — იერარქია (სექცია 15): charcoal / charcoal-70 / charcoal-45 / taupe.
- **hover** — #B8AA98 ყველა კლიკად ელემენტზე (სექცია 5).
- **ფოტოები** — `.ph` + `.slot-fill` + სწორი aspect-ratio (სექცია 14).
- **კომპონენტები** — ღილაკები (13), byline (12.8), drop cap (12.9), Newsletter/Footer (11), ზუსტი სპეცით.
- **responsive** — სავალდებულო (სექცია 10): desktop + ტაბლეტი + მობილური.

### ნაბიჯი 3: შეამოწმე
- გახსენი **Edge InPrivate**-ში (ქეშის გარეშე).
- შეამოწმე **3 ზომაზე**: Desktop / iPad / iPhone.

### ნაბიჯი 4: დაადასტურე
მითხარი მოკლედ: რა გააკეთე და დიზაინ-სისტემის რომელ წესებს დაეყრდენი.
